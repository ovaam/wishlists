# Отладка синхронизации Firebase

## Проверка работы Firebase

### Шаг 1: Откройте консоль браузера

1. Откройте сайт в браузере
2. Нажмите F12 (или Cmd+Option+I на Mac)
3. Перейдите на вкладку "Console"

### Шаг 2: Проверьте сообщения

Должны быть сообщения:
- ✅ `Firebase подключен, данные синхронизируются`
- ✅ `Данные получены из Firebase: {...}`
- ✅ `Данные сохранены в Firebase`

Если видите ошибки:
- ❌ `Firebase не подключен` - проверьте подключение SDK
- ❌ `Permission denied` - проверьте правила безопасности
- ❌ `Network error` - проверьте интернет-соединение

### Шаг 3: Проверьте правила безопасности в Firebase

1. Откройте https://console.firebase.google.com
2. Выберите проект `wishlist-8d5ee`
3. Realtime Database → Rules
4. Должны быть правила:
```json
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
```
5. Нажмите "Publish"

### Шаг 4: Проверьте данные в Firebase Console

1. Realtime Database → Data
2. Должна быть структура:
```
wishlists
  ├── леся: [...]
  ├── мама: [...]
  └── алина: [...]
```

### Шаг 5: Тест синхронизации

1. Откройте сайт на двух устройствах/вкладках
2. На первом устройстве добавьте желание
3. На втором устройстве обновите страницу (F5)
4. Желание должно появиться автоматически

## Если не работает

1. **Очистите кеш браузера** (Ctrl+Shift+Del или Cmd+Shift+Del)
2. **Проверьте, что Firebase SDK загружается** - в консоли не должно быть ошибок загрузки скриптов
3. **Проверьте network tab** - должны быть запросы к `firebaseio.com`
4. **Попробуйте в режиме инкогнито** - чтобы исключить проблемы с расширениями

## Альтернатива: Проверка через Firebase Console

Если данные не синхронизируются, можно вручную проверить:
1. Откройте Firebase Console → Realtime Database
2. Добавьте тестовые данные вручную
3. Они должны появиться на сайте
